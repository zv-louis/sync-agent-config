# sync-mcp-config

[English](README.md) | 日本語

異なるAgentツール間でMCP（Model Context Protocol）サーバー設定を同期するためのスキルです。

## 概要

このプロジェクトは、Claude Code、VSCode、Claude Desktop、Codex、Gemini CLIなど、異なるAgent環境間でMCPサーバー構成を同期できるスキルを提供します。
ソースAgentツールからMCP構成を読み取り、ターゲットAgentツールに適用することで、複数の開発環境で一貫した設定を確保します。

## 機能

- **クロスプラットフォーム対応**:
  Windows、macOS、Linuxで動作します
- **マルチツール互換性**:
  以下を含む様々なAgentツールをサポート:
  - Claude Code
  - VSCode
  - Claude Desktop
  - Codex CLI
  - Gemini CLI
  - Antigravity
- **安全な同期**:
  変更を適用する前にバックアップを作成し、構成を検証します
- **インテリジェントな処理**:
  異なるAgentツール間のフォーマットの違いを自動的に処理します
- **Web検索統合**:
  ドキュメントが利用できない場合、自動的に構成仕様を検索します

## インストール

1. `dist`ディレクトリから最新リリースをダウンロードします:

   ```txt
   sync-mcp-config-<version>.zip
   ```

2. お使いのAgentツールのスキルインストール手順に従ってインストールします。

## 使用方法

Agentにツール間でMCP設定を同期するよう指示するだけです:

**例:**

- 「Claude CodeのMCP設定をCodexに適用したい」
- 「VSCodeからClaude DesktopにMCP構成を同期して」
- 「Claude DesktopからClaude CodeにMCPサーバー設定をコピーして」

スキルは以下を実行します:

1. オペレーティングシステム環境を確認
2. ソースAgentツールのMCP構成を読み取り
3. 利用可能なMCPサーバーをチェック
4. ターゲットAgentツールに設定を適用
5. 構成を検証

## 動作の仕組み

1. **OS環境チェック**:
  オペレーティングシステムを検出し、適切なパスを設定
2. **構成の検証**:
  ソースとターゲットの両方の構成フォーマットを読み取り理解
3. **サーバー検出**:
  ソース構成に登録されているすべてのMCPサーバーを識別
4. **フォーマット変換**:
  ターゲットツールのフォーマットに合わせて自動的に設定を変換
5. **安全な適用**:
  変更を加える前に既存の構成をバックアップ
6. **検証**:
  新しい構成が有効であることを確認

## 構成ファイルの場所

スキルは、オペレーティングシステムとAgentツールに基づいて構成ファイルのパスを自動的に検出します。サポートされている各Agentツールのリファレンスドキュメントは、[`sync-mcp-config/reference/`](sync-mcp-config/reference/)ディレクトリにあります。

## 開発

### プロジェクト構造

```txt
sync-mcp-config/
├── SKILL.md              # スキル定義と手順
├── VERSION               # 現在のバージョン番号
└── reference/            # 構成リファレンスドキュメント
    ├── antigravity.md
    ├── claude_code.md
    ├── claude_desktop.md
    ├── codex_cli.md
    ├── gemini_cli.md
    └── vscode.md
```

### リリースのビルド

リリースアーカイブを作成するには:

1. スキルアーカイブには、`sync-mcp-config`ディレクトリ内のすべてのファイルとサブディレクトリを含める必要があります
2. アーカイブ名を`sync-mcp-config-<version>.zip`とします。`<version>`は[`sync-mcp-config/VERSION`](sync-mcp-config/VERSION)ファイル内のバージョンと一致させます
3. zipファイルをこのリポジトリのルートにある`dist`ディレクトリに出力します

## 安全機能

- **バックアップ作成**:
  変更を加える前に既存の構成を自動的にバックアップ
- **競合検出**:
  サーバーが既に登録されている場合を識別し、ユーザーの確認を求めます
- **検証**:
  変更後に構成ファイルの整合性をチェック
- **読み取り専用モード**:
  MCPサーバー設定のみを変更し、他の構成には触れません

## エラーハンドリング

スキルは様々なエラーシナリオを処理します:

- ソース構成ファイルの欠落
- 書き込み保護されたターゲットファイル
- 無効な構成フォーマット
- 異なるAgentツール間での互換性のない設定
